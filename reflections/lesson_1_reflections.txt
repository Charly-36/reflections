
¿De qué manera la visualización de una diferencia entre dos versiones de un archivo lo ayudó a ver el error que
¿Fue presentado?

    Al usar el comando FC, que nos ayuda a comparar dos archivos,nos muestra en qué línea son diferentes

¿Cómo podría tener un acceso fácil a todo el historial de un archivo hacer que usted tenga más acceso programador eficiente a largo plazo?

    Que haya en cada archivo,un historial completo de las modificaciones que se han hecho, mostrando siempre la última versión modificada.


¿Cuáles crees que son los pros y los contras de elegir manualmente cuándo crear un
confirme, como lo hace en Git, vs tener versiones guardadas automáticamente, como Google
docs hace?
    Los pros,serían guardar, respaldar la información cada vez que sea ha hecho algo importante, haya sido una o muchas lineas pero que son muy importantes.
    Contras, que si son importantes y son a cada rato,obtendremos demasiadas versiones cuando tal vez no sea necesario o tan eficiente.


¿Por qué cree que algunos sistemas de control de versiones, como Git, permiten guardar múltiples
archivos en una confirmación, mientras que otros, como Google Docs, tratan cada archivo por separado?

    Para evitar el uso mayor de memoria creando varios archivos con diferentes versiones.

¿Cómo puedes usar los comandos git log y git diff para ver el historial de archivos?

    git log: El comportamiento predeterminado es enumerar todas las confirmaciones en la rama actual en orden cronológico inverso, incluidos detalles como el autor, la hora y el mensaje.
    git diff: El comportamiento predeterminado es enumerar todos los cambios que no están en escena para la próxima confirmación. 

¿De qué manera el uso del control de versiones puede hacer que tenga más confianza para realizar cambios,
podría romper algo?
    Podría ser que cada vez que se escribe un nuevo caracter pueda crear una nueva versión, así es más fácil recuperaralgo pero el problema sería que habría cientos de versiones.


Ahora que ya tiene configurada su área de trabajo, ¿para qué quiere intentar usar Git?

    Para trabajar los proyectos o trabajos en equipo, en especial para programar con los miembros de un equipo.
